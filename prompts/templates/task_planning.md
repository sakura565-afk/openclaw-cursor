## Task Planning Template

### Purpose
Use this template to convert an ambiguous request into an executable technical plan.

### Inputs
- Request summary:
- Business/user outcome:
- Known constraints:
- Dependencies (systems/teams/tools):
- Definition of done:

### Required Structure
1. **Goal and Scope**
   - Clarify in-scope and out-of-scope work.
2. **Assumptions**
   - State explicit assumptions and unknowns.
3. **Work Breakdown**
   - Split work into concrete implementation tasks.
4. **Decision Points**
   - Note architectural or product decisions required.
5. **Risk Register**
   - Document major delivery and quality risks.
6. **Verification Strategy**
   - Explain how each task will be validated.
7. **Handoff Notes**
   - Include context needed for another engineer/agent.

### Output Format
Return markdown with these headings:
- Goal and Scope
- Assumptions
- Work Breakdown
- Decision Points
- Risk Register
- Verification Strategy
- Handoff Notes
