## Skill Documentation Template

### Purpose
Use this template to document a reusable engineering skill, workflow, or playbook.

### Inputs
- Skill name:
- Target audience (role and level):
- Prerequisites:
- Required tools/resources:
- Expected outcomes:

### Required Structure
1. **Skill Overview**
   - Define what the skill is and when to use it.
2. **Prerequisites**
   - List technical and domain prerequisites.
3. **Core Procedure**
   - Step-by-step procedure with decision points.
4. **Examples**
   - Provide at least one realistic example.
5. **Failure Modes**
   - Common mistakes and how to avoid them.
6. **Verification**
   - How to validate successful execution.
7. **Reference Links**
   - Internal docs, code paths, and external references.

### Output Format
Return markdown with these headings:
- Skill Overview
- Prerequisites
- Core Procedure
- Examples
- Failure Modes
- Verification
- Reference Links
