## Implementation Template

### Purpose
Use this template to execute a scoped coding task with clear, testable outputs.

### Inputs
- Objective:
- Scope (in/out):
- Affected files/components:
- Acceptance criteria:
- Constraints (performance, security, style):

### Required Structure
1. **Implementation Goal**
   - State the intended behavior change.
2. **Planned Changes**
   - Enumerate files and the specific edits to make.
3. **Execution Steps**
   - Ordered list of implementation steps.
4. **Edge Cases**
   - Identify expected edge behavior.
5. **Testing**
   - Unit/integration/manual checks to run.
6. **Completion Checklist**
   - Criteria that must be true before finishing.

### Output Format
Return markdown with these headings:
- Implementation Goal
- Planned Changes
- Execution Steps
- Edge Cases
- Testing
- Completion Checklist
