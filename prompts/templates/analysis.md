## Analysis Template

### Purpose
Use this template to break down a problem before implementation.

### Inputs
- Task description:
- Relevant files/modules:
- Constraints (technical, product, performance, security):
- Known unknowns:

### Required Structure
1. **Problem Statement**
   - Summarize what must be solved in 2-4 sentences.
2. **Current State**
   - Describe current behavior and limitations.
3. **Root Cause / Gaps**
   - Identify technical causes or missing pieces.
4. **Solution Options**
   - List 2-3 feasible approaches with trade-offs.
5. **Recommended Approach**
   - Explain why it is preferred.
6. **Risks and Mitigations**
   - Include regression, compatibility, and test risks.
7. **Validation Plan**
   - Define how to verify correctness.

### Output Format
Return markdown with these exact headings:
- Problem Statement
- Current State
- Root Cause / Gaps
- Solution Options
- Recommended Approach
- Risks and Mitigations
- Validation Plan
