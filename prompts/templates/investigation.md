Use this template when the goal is to understand behavior, trace a flow, or answer “how / why / where” without necessarily shipping a fix in the same step.

## Boundaries

## Output format

1. **Direct answer** to the question (or clearest partial answer with gaps).
2. **Evidence** — file references, function names, commands used, key snippets (cite paths).
3. **Follow-ups** — optional next experiments or changes, prioritized.
